## 1. プロジェクト概要

### 1.1 プロジェクト名
TODOタスク管理アプリケーション

### 1.2 プロジェクトの目的
個人またはチームがタスクを効率的に管理できるWebアプリケーションを提供する。タスクの作成、閲覧、更新、削除（CRUD操作）および検索機能を通じて、ユーザーの生産性向上を支援する。

### 1.3 プロジェクトスコープ

#### 対象範囲（In Scope）
- タスクの基本的なCRUD操作
- タスクのステータス管理（TODO/DOING/DONE）
- タスクの検索機能（概要、ステータス）
- Webブラウザベースのユーザーインターフェース
- H2インメモリデータベースを使用したデータ永続化

#### 対象外（Out of Scope）
- ユーザー認証・認可機能
- マルチユーザー対応
- タスクの優先度管理
- タスクの期限管理
- タスクの担当者割り当て
- タスクの添付ファイル機能
- 通知機能
- REST API公開
- モバイルアプリケーション

### 1.4 ステークホルダー

| 役割 | 責任 |
|------|------|
| プロダクトオーナー | 要件定義、優先順位決定、受入テスト |
| 開発チーム | 設計、実装、単体テスト、結合テスト |
| QAチーム | テスト計画、テスト実行、バグ報告 |
| エンドユーザー | システム利用、フィードバック提供 |

### 1.5 成功基準
- すべての機能要件が実装され、テストに合格すること
- レスポンスタイムが非機能要件を満たすこと
- バグ0件でリリースできること
- ユーザビリティテストで80%以上の満足度を得ること

#### F-001: タスク一覧表示

**機能概要**
登録されているすべてのタスクを一覧表示する。

**アクセス方法**
- URL: `GET /tasks`
- ナビゲーション: ホーム画面の「タスク管理へ」リンク

**表示項目**
| 項目 | 表示形式 | 備考 |
|------|----------|------|
| ID | リンク | クリックでタスク詳細画面へ遷移 |
| 概要 | リンク | クリックでタスク詳細画面へ遷移 |
| ステータス | テキスト | TODO/DOING/DONEのいずれか |

**操作要素**
| 要素 | 種別 | 動作 |
|------|------|------|
| 作成ボタン | ボタン | タスク作成フォーム画面へ遷移 |
| 検索フォーム | フォーム | タスク検索を実行（F-002） |
| タスクID/概要リンク | リンク | タスク詳細画面へ遷移（F-003） |

**ビジネスルール**
- タスクはID順に表示される
- タスクが0件の場合、テーブルのヘッダーのみ表示される
- 削除済みタスクは表示されない

**画面遷移**
```
一覧画面 → 作成ボタン → 作成フォーム画面（F-004）
一覧画面 → タスクリンク → 詳細画面（F-003）
一覧画面 → 検索実行 → 一覧画面（検索結果）（F-002）
```

#### F-002: タスク検索

**機能概要**
概要とステータスを条件としてタスクを検索し、一覧表示する。

**アクセス方法**
- URL: `GET /tasks?summary={keyword}&status={status1}&status={status2}...`
- 操作: タスク一覧画面の検索フォームから実行

**検索条件**

| 項目 | 型 | 必須 | 検索方式 | 備考 |
|------|-----|------|----------|------|
| 概要 | String | × | 部分一致（LIKE） | 空白の場合は条件なし |
| ステータス | List<Enum> | × | 完全一致（IN句） | 複数選択可能、未選択の場合は条件なし |

**検索ロジック**
```sql
SELECT id, summary, description, status
FROM tasks
WHERE
  (summary LIKE '%{keyword}%' OR keyword IS NULL)
  AND
  (status IN ({status1}, {status2}, ...) OR status IS NULL)
```

**ビジネスルール**
- すべての検索条件が未入力の場合、全件表示される
- 概要は大文字小文字を区別しない
- 検索結果が0件の場合、テーブルヘッダーのみ表示される
- 検索実行後、検索条件は保持される（フォームに残る）

**表示要件**
- 検索結果はF-001と同じ形式で表示される
- 検索フォームには実行した検索条件が保持される（チェックボックスの状態含む）

**アクセス方法**

| 概要 | テキスト | 最大256文字 |
| 詳細 | テキスト（複数行） | 改行を含む |
| ステータス | テキスト | TODO/DOING/DONEのいずれか |

**操作要素**
| 要素 | 種別 | 動作 |
|------|------|------|
| 一覧へ戻るボタン | ボタン | タスク一覧画面へ遷移 |

#### F-004: タスク新規作成

**機能概要**
新しいタスクを作成し、データベースに保存する。

**アクセス方法**
- URL: `GET /tasks/creationForm` （フォーム表示）
- URL: `POST /tasks` （作成実行）
- 操作: タスク一覧画面の「作成」ボタンをクリック

**入力項目**

| 項目 | 型 | 必須 | 最大長 | 初期値 | 備考 |
|------|-----|------|--------|--------|------|
| 概要 | String | ○ | 256文字 | 空 | タスクの簡潔な説明 |
| 詳細 | String | × | 無制限 | 空 | タスクの詳細説明、改行可能 |
| ステータス | Enum | ○ | - | TODO | TODO/DOING/DONEから選択 |

**バリデーションルール**

| 項目 | ルール | エラーメッセージ |
|------|--------|------------------|
| 概要 | 必須入力 | "概要は必須です" |
| 概要 | 最大256文字 | "256文字以内で入力してください" |
| 概要 | 空白のみ不可 | "概要は必須です" |
| ステータス | 必須選択 | "ステータスを選択してください" |
| ステータス | TODO/DOING/DONEのいずれか | "TODO, DOING, DONE のいずれかを選択してください" |

**処理フロー**
1. ユーザーが「作成」ボタンをクリック
2. 作成フォーム画面が表示される（mode=CREATE）
3. ユーザーが情報を入力し「保存」ボタンをクリック
4. サーバー側でバリデーション実行
5. バリデーションエラーがある場合
   - エラーメッセージを表示し、入力値を保持したまま作成フォーム画面を再表示
6. バリデーション成功の場合
   - TaskEntityを生成（IDは自動採番）
   - データベースにINSERT
   - タスク一覧画面へリダイレクト

**データベース操作**
```sql
INSERT INTO tasks (summary, description, status)
VALUES (?, ?, ?)
```

**ビジネスルール**
- IDは自動採番される（AUTO_INCREMENT）
- 作成日時は記録されない（将来拡張可能）
- 作成後、タスク一覧画面にリダイレクトされる

**画面遷移**
```
一覧画面 → 作成ボタン → 作成フォーム画面
作成フォーム画面 → キャンセル → 一覧画面
作成フォーム画面 → 保存（バリデーションエラー） → 作成フォーム画面（エラー表示）
作成フォーム画面 → 保存（成功） → 一覧画面（リダイレクト）
```

#### F-005: タスク編集

**機能概要**
既存タスクの情報を編集し、データベースに保存する。

**アクセス方法**
- URL: `GET /tasks/{id}/editForm` （フォーム表示）
- URL: `PUT /tasks/{id}` （更新実行）
- 操作: タスク詳細画面の「編集」ボタンをクリック

**入力項目**
F-004と同じ

**バリデーションルール**
F-004と同じ

**処理フロー**
1. ユーザーが詳細画面の「編集」ボタンをクリック
2. 指定されたIDのタスクをデータベースから取得
3. タスクが存在しない場合、404エラーページを表示
4. タスクが存在する場合、編集フォーム画面が表示される（mode=EDIT）
5. フォームに既存データがプリセットされる
6. ユーザーが情報を編集し「更新」ボタンをクリック
7. サーバー側でバリデーション実行
8. バリデーションエラーがある場合
   - エラーメッセージを表示し、入力値を保持したまま編集フォーム画面を再表示
9. バリデーション成功の場合
   - TaskEntityを生成（IDは変更前と同じ）
   - データベースをUPDATE
   - タスク詳細画面へリダイレクト

**データベース操作**
```sql
UPDATE tasks
SET summary = ?, description = ?, status = ?
WHERE id = ?
```

**ビジネスルール**
- IDは変更できない
- 存在しないIDを指定した場合は404エラー
- 更新後、タスク詳細画面にリダイレクトされる
- 更新日時は記録されない（将来拡張可能）

**エラー処理**
| エラー条件 | 動作 |
|-----------|------|
| 指定されたIDのタスクが存在しない | 404エラーページを表示 |
| IDが数値でない | 400エラー |
| バリデーションエラー | エラーメッセージを表示し、フォームを再表示 |

**画面遷移**
```
詳細画面 → 編集ボタン → 編集フォーム画面
編集フォーム画面 → キャンセル → 詳細画面
編集フォーム画面 → 更新（バリデーションエラー） → 編集フォーム画面（エラー表示）
編集フォーム画面 → 更新（成功） → 詳細画面（リダイレクト）
```

**技術的考慮事項**
- HTTPメソッドはPUTを使用
- HTMLフォームではPUTメソッドを直接サポートしていないため、HiddenHttpMethodFilterを使用
  ```html
  <input type="hidden" name="_method" value="PUT">
  ```
- `spring.mvc.hiddenmethod.filter.enabled=true` の設定が必要

#### F-006: タスク削除

**機能概要**
既存タスクをデータベースから削除する。

**アクセス方法**
- URL: `DELETE /tasks/{id}`
- 操作: タスク詳細画面の「削除」ボタンをクリック

**処理フロー**
1. ユーザーが詳細画面の「削除」ボタンをクリック
2. サーバー側で指定されたIDのタスクを削除
3. タスク一覧画面へリダイレクト

**データベース操作**
```sql
DELETE FROM tasks WHERE id = ?
```

**ビジネスルール**
- 削除は論理削除ではなく物理削除
- 削除確認ダイアログは表示されない（将来拡張可能）
- 存在しないIDを指定しても正常終了する（エラーにならない）
- 削除後、タスク一覧画面にリダイレクトされる

**エラー処理**
| エラー条件 | 動作 |
|-----------|------|
| IDが数値でない | 400エラー |
| 存在しないIDを指定 | 正常終了（エラーにしない） |

**画面遷移**
```
詳細画面 → 削除ボタン → 一覧画面（リダイレクト）
```

**技術的考慮事項**
- HTTPメソッドはDELETEを使用
- HTMLフォームではDELETEメソッドを直接サポートしていないため、HiddenHttpMethodFilterを使用
  ```html
  <input type="hidden" name="_method" value="DELETE">
  ```

**将来拡張要件**
- 削除確認ダイアログの追加
- 論理削除への変更
- 削除権限チェック

#### F-008: エラーページ表示

**機能概要**
エラー発生時に適切なエラーページを表示する。

**エラーの種類**

| HTTPステータス | エラー名 | 表示ページ | 発生条件 |
|---------------|---------|-----------|---------|
| 404 | Not Found | 404専用ページ | リソースが見つからない場合 |
| 400 | Bad Request | 汎用エラーページ | リクエストが不正な場合 |
| 500 | Internal Server Error | 汎用エラーページ | サーバー内部エラー |

**404エラーページ**
- 専用テンプレート: `templates/error/404.html`
- 表示内容: 「ページが見つかりません」メッセージ
- トップへ戻るリンク

**汎用エラーページ**
- テンプレート: `templates/error.html`
- 表示内容: エラーメッセージ、ステータスコード
- トップへ戻るリンク

**ビジネスルール**
- すべてのエラーは適切なHTTPステータスコードで応答する
- エラー詳細はログに出力する
- 本番環境ではスタックトレースを表示しない（開発環境のみ）

## 4. 非機能要件

### 4.1 パフォーマンス要件

| 項目 | 要件 | 測定方法 |
|------|------|----------|
| レスポンスタイム | 画面表示は2秒以内 | ブラウザDevTools計測 |
| 同時接続数 | 単一ユーザー想定（開発環境） | - |
| データ量 | タスク数1000件まで正常動作 | 負荷テスト |

### 4.2 セキュリティ要件

| 項目 | 要件 | 実装方法 |
|------|------|----------|
| SQLインジェクション対策 | 必須 | MyBatisのプレースホルダー使用 |
| XSS対策 | 必須 | Thymeleafの自動エスケープ |
| CSRF対策 | 現バージョンでは対象外 | 将来実装予定 |
| 入力バリデーション | 必須 | Spring Validation使用 |
| HTTPSサポート | 開発環境では対象外 | 本番では必須 |

**セキュリティ実装詳細**
- すべてのSQL文はプリペアドステートメント（プレースホルダー）を使用
- HTMLテンプレートはThymeleafの自動エスケープ機能を使用（`th:text`, `th:utext`の使い分け）
- すべてのユーザー入力に対してサーバー側バリデーションを実行
- クライアント側バリデーションは補助的な位置づけ

### 4.3 可用性要件

| 項目 | 要件 |
|------|------|
| 稼働率 | 開発環境のため規定なし |
| 障害復旧 | アプリケーション再起動で復旧（データは失われる） |
| バックアップ | 対象外（インメモリDB） |

### 4.4 保守性要件

| 項目 | 要件 |
|------|------|
| コーディング規約 | Javaコーディング規約に準拠 |
| コードレビュー | 必須 |
| ドキュメント | コード内Javadoc、README、本要件定義書 |
| ロギング | エラーログ、SQLログを出力 |

**ログ出力仕様**

### 5.2 物理データモデル

#### tasksテーブル

**テーブル名**: `tasks`

**テーブル説明**: TODOタスクを管理するテーブル

| カラム名 | 型 | NOT NULL | デフォルト値 | キー | 説明 |
|---------|-----|----------|-------------|------|------|
| id | BIGINT | ○ | AUTO_INCREMENT | PK | タスクID（自動採番） |
| summary | VARCHAR(256) | ○ | - | - | タスク概要 |
| description | TEXT | × | NULL | - | タスク詳細説明 |
| status | VARCHAR(256) | ○ | - | - | ステータス（TODO/DOING/DONE） |

**制約条件**
```sql
-- 主キー制約
PRIMARY KEY (id)

-- NOT NULL制約
summary NOT NULL
status NOT NULL

-- CHECK制約（実装推奨、現バージョンでは未実装）
CHECK (status IN ('TODO', 'DOING', 'DONE'))

-- 文字数制約
summary: 最大256文字
description: 無制限（TEXT型）
status: 最大256文字（実質的には10文字以内）
```

**インデックス**
- 現バージョンではPRIMARY KEY以外のインデックスなし
- 将来的にstatusカラムへのインデックス追加を検討

**DDL**
```sql
CREATE TABLE tasks (
  id BIGINT NOT NULL PRIMARY KEY AUTO_INCREMENT,
  summary VARCHAR(256) NOT NULL,
  description TEXT,
  status VARCHAR(256) NOT NULL
);
```

## 8. バリデーション仕様

### 8.1 バリデーション実装方針

- すべてのバリデーションはサーバー側で実行する（セキュリティ上の理由）
- クライアント側バリデーション（HTML5 validation）は補助的に使用可能
- Spring Validationフレームワークを使用
- バリデーションエラー時は入力値を保持したままフォームを再表示

### 8.2 バリデーションルール詳細

#### TaskFormバリデーション

**対象**: タスク作成・編集

| フィールド | アノテーション | パラメータ | エラーメッセージ | 説明 |
|-----------|--------------|-----------|-----------------|------|
| summary | `@NotBlank` | - | "概要は必須です" | 必須入力チェック |
| summary | `@Size` | max=256 | "256文字以内で入力してください" | 最大文字数チェック |
| description | - | - | - | 任意項目（バリデーションなし） |
| status | `@NotBlank` | - | "ステータスを選択してください" | 必須選択チェック |
| status | `@Pattern` | regexp="TODO\|DOING\|DONE" | "TODO, DOING, DONE のいずれかを選択してください" | パターンマッチング |

**実装例（TaskForm.java）**
```java
public record TaskForm(
    @NotBlank(message = "概要は必須です")
    @Size(max = 256, message = "256文字以内で入力してください")
    String summary,

    String description,

    @NotBlank(message = "ステータスを選択してください")
    @Pattern(regexp = "TODO|DOING|DONE", message = "TODO, DOING, DONE のいずれかを選択してください")
    String status
) { ... }
```

**コントローラー側の処理**
```java
@PostMapping
public String create(@Validated TaskForm form, BindingResult bindingResult, Model model) {
    if (bindingResult.hasErrors()) {
        // バリデーションエラーがある場合、フォームを再表示
        model.addAttribute("mode", "CREATE");
        return "tasks/form";
    }
    // バリデーション成功時の処理
    taskService.create(form.toEntity());
    return "redirect:/tasks";
}
```

### 8.3 バリデーションエラー表示仕様

```java
public class TaskNotFoundException extends RuntimeException {
    public TaskNotFoundException() {
        super("タスクが見つかりません");
    }
}
```

**使用例**
```java
@GetMapping("/{id}")
public String showDetail(@PathVariable("id") long taskId, Model model) {
    var taskDTO = taskService.findById(taskId)
            .map(TaskDTO::toDTO)
            .orElseThrow(TaskNotFoundException::new);
    model.addAttribute("task", taskDTO);
    return "tasks/detail";
}
```

**処理フロー**
1. `TaskNotFoundException`がスローされる
2. Spring Bootが自動的に404エラーページにマッピング
3. `templates/error/404.html`が表示される

### 9.3 エラーページ仕様

#### 404エラーページ (`templates/error/404.html`)
- カスタムメッセージ: "ページが見つかりません"
- トップへ戻るリンク
- ユーザーフレンドリーなデザイン

#### 汎用エラーページ (`templates/error.html`)
- エラーコード表示
- エラーメッセージ表示（本番環境では詳細非表示推奨）
- トップへ戻るリンク

### 9.4 ロギング仕様

| ログレベル | 対象 | 出力内容 |
|----------|------|---------|
| ERROR | すべての例外 | スタックトレース、エラーメッセージ |
| WARN | 非致命的なエラー | 警告メッセージ |
| INFO | 重要な処理 | 処理開始/終了、主要な状態変化 |
| DEBUG | SQL実行ログ | MyBatis SQLログ（開発時のみ） |

**設定（application.properties）**
```properties
# SQLログ出力（開発時）
logging.level.com.example.todo.repository=DEBUG

# 本番環境では以下を推奨
# logging.level.com.example.todo.repository=INFO
```

### 10.4 システムテスト要件

**テストシナリオ**

1. **タスク作成シナリオ**
   - ホーム画面からタスク一覧へ遷移
   - 作成ボタンをクリック
   - フォームに入力し保存
   - 一覧画面で作成されたタスクを確認

2. **タスク検索シナリオ**
   - タスク一覧画面で検索条件を入力
   - 検索実行
   - 結果を確認
   - 検索条件が保持されていることを確認

3. **タスク編集シナリオ**
   - タスク詳細画面へ遷移
   - 編集ボタンをクリック
   - フォームで内容を変更し更新
   - 詳細画面で変更が反映されていることを確認

4. **タスク削除シナリオ**
   - タスク詳細画面へ遷移
   - 削除ボタンをクリック
   - 一覧画面で削除されていることを確認

5. **バリデーションエラーシナリオ**
   - 作成フォームで概要を空にして保存
   - エラーメッセージが表示されることを確認
   - 入力値が保持されていることを確認

6. **404エラーシナリオ**
   - 存在しないタスクIDで詳細画面にアクセス
   - 404エラーページが表示されることを確認
